# 変更履歴

このプロジェクトのすべての注目すべき変更はこのファイルに記録されます。

フォーマットは [Keep a Changelog](https://keepachangelog.com/ja/1.0.0/) に基づいており、
このプロジェクトは [セマンティックバージョニング](https://semver.org/lang/ja/) に準拠しています。

## [1.4.0] - 2025-07-23

### 追加

- 新モジュール「デイリーチェックリスト」: 進捗を最大化するための完全な日課リスト
- チェック状態を安定したIDで保存（テキスト修正や新規追加に対応）
- 08:00に自動デイリーリセット（確認バナー付き）
- 確認付き手動リセットボタン
- DemiDevimonループトグル（リスト内）、設定可能なチケット数でミッションサイクルを繰り返し
- インラインタグバッジ（ミッション、リソース消費、ショップ、キャンプ、PvP）
- 「完了を非表示」トグル（デフォルトON）でリストをすっきり保つ
- 完全なi18nサポート: 全ステップとタグが6言語に翻訳
- `<html lang>` のMutationObserverで言語変更時に再レンダリング

## [1.3.0] - 2025-07-21

### 追加

- ガチャレベル: 「未回収の時間」入力（HH:MM:SS）、最終回収タイムスタンプを追跡
- ガチャレベル: 「更新」ボタンで最終回収時刻を設定、再計算時に経過時間が自動更新
- ガチャレベル: 「最終回収: HH:MM」を参考表示
- ガチャレベル: パッシブ蓄積が満杯（≥8h）の場合に回収リマインダー表示、次の推奨回収時刻（+7h）を表示
- ガチャレベル: 現在のデジエメラルドに+1、+10、+50、+100のクイックボタン追加
- ガチャ計算で既に蓄積されたエメラルドを考慮（パッシブ上限8h）

## [1.2.0] - 2025-07-21

### 追加

- パッシブ獲得: ガチャレベルアップを計算する新しい「ガチャレベル」タブ
- カードガチャとサポートガチャ、各引いた回数/目標/残り/現在のチケット
- メカニクス: マルチ1回に30チケット（35回ガチャ）、チケット1枚=20デジエメラルド
- マルチ回数、チケットコスト、デジエメラルドコスト、パッシブ時間を計算

### 変更

- パッシブ獲得: 3つのサブタブに分割（待機時間、エメラルド、ガチャレベル）
- 報酬設定はすべてのタブで常に表示
- 計算ボタンを押すまで結果パネルは非表示（全3タブ）
- フロアが未設定の場合、警告を表示し計算をブロック
- 計算ボタンを全幅に変更

## [1.1.0] - 2025-07-21

### 改善

- Memory Helper: ボード上に既に2枚配置されているデジモンはセレクターで暗く表示され、選択できなくなります

### 追加

- Docker のビルド、プッシュ、リモートデプロイを自動化する `deploy.sh` スクリプト

### 変更

- チェンジログモジュールが `/changelogs/{locale}.md` から言語別のチェンジログを読み込むようになりました
- サポートされている6言語すべてでチェンジログが利用可能（en, es, it, pt, de, ja）
- 重複していた `public/CHANGELOG.md` を削除し、言語別ファイルに移行

## [1.0.0] - 2025-01-20

### 追加

- App Shellとサイドバーナビゲーションによるマルチユーティリティアーキテクチャ
- 外部依存関係のない静的Node.jsサーバー
- 6言語をサポートする国際化システム（i18n）
- 拡張性のためのセントラルモジュールレジストリ
- Memory Game Helperモジュール（元のモノリスから抽出）
- パッシブゲイン計算モジュール（新しいユーティリティ）
- ローカルデジモン画像（12 PNG）
- モバイルで折りたたみ可能なサイドバーを備えたレスポンシブデザイン
- チェンジログ付きバージョニングシステム

### 変更

- モノリシックアプリケーション（単一HTML）からモジュラーアーキテクチャへのリファクタリング
- 外部画像からローカルファイルへの移行
